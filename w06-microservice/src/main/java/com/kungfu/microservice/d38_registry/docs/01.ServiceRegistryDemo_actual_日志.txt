========================================
  手写 Mini 注册中心
========================================

=== 一、服务注册发现原理 ===

  ┌─────────────────────────────────────────────────────────────┐
  │                    注册中心 (Nacos/Eureka)                    │
  │  ┌─────────────────────────────────────────────────────────┐│
  │  │ 注册表: {                                               ││
  │  │   "order-service": [192.168.1.1:8080, 192.168.1.2:8080] ││
  │  │   "stock-service": [192.168.1.3:8081]                   ││
  │  │ }                                                       ││
  │  └─────────────────────────────────────────────────────────┘│
  └──────────────────────────┬──────────────────────────────────┘
           ↑ 注册/心跳        ↓ 发现/订阅
  ┌────────────────┐    ┌────────────────┐
  │ 服务提供者     │    │ 服务消费者     │
  │ (order-service)│    │ (api-gateway)  │
  └────────────────┘    └────────────────┘

  核心流程：
    1. 服务启动 → 向注册中心注册（IP + 端口 + 元数据）
    2. 消费者 → 从注册中心获取服务实例列表
    3. 服务定时发送心跳（默认 5s/次）
    4. 注册中心检测心跳超时 → 剔除实例
    5. 实例变更 → 推送通知消费者（或消费者定时拉取）

=== 二、模拟注册发现流程 ===

  Step 1: 注册服务实例
    [注册] order-service#192.168.1.1:8080 注册成功
    [注册] order-service#192.168.1.2:8080 注册成功
    [注册] order-service#192.168.1.3:8080 注册成功

  Step 2: 服务发现
    发现 order-service 实例: [order-service#192.168.1.1:8080, order-service#192.168.1.2:8080, order-service#192.168.1.3:8080]
    实例数: 3

  Step 3: 心跳续约
    order1, order2 续约成功
    order3 未续约（模拟宕机）

  Step 4: 等待心跳超时（6 秒）...
  Step 5: 再次发现 order-service
    剩余实例: [order-service#192.168.1.1:8080, order-service#192.168.1.2:8080, order-service#192.168.1.3:8080]
    → order3 因心跳超时被剔除

=== 三、注册中心对比 ===

  ┌──────────────┬──────────────┬──────────────┬──────────────┬──────────────┐
  │              │ Nacos ★     │ Eureka       │ ZooKeeper    │ Consul       │
  ├──────────────┼──────────────┼──────────────┼──────────────┼──────────────┤
  │ CAP          │ AP + CP 可选 │ AP           │ CP           │ CP           │
  │ 健康检查     │ TCP/HTTP/SQL │ 心跳         │ 长连接       │ TCP/HTTP     │
  │ 推送方式     │ 长轮询+推送  │ 定时拉取     │ Watch 推送   │ 长轮询       │
  │ 配置中心     │ ✓ 内置       │ ✗            │ ✗            │ ✓ KV Store  │
  │ 雪崩保护     │ ✓            │ ✓ 自我保护   │ ✗            │ ✗            │
  │ 适用场景     │ 阿里系/通用  │ Netflix 系   │ Dubbo 系     │ HashiCorp 系│
  └──────────────┴──────────────┴──────────────┴──────────────┴──────────────┘

  ★ 面试必背：
    Q: Nacos 和 Eureka 的区别？
    A: 1) Nacos 支持 AP+CP 切换，Eureka 只有 AP
       2) Nacos 有配置中心功能，Eureka 没有
       3) Nacos 用长轮询+推送，Eureka 用定时拉取（30s）
       4) Nacos 支持临时实例(AP)+持久实例(CP)
