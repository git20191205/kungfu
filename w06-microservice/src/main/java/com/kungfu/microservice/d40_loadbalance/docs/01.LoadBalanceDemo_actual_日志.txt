=== 二、五大算法 10000 次请求分布 ===

  ── Round Robin (轮询) ──
    192.168.1.1:8080      3334 ( 33.3%) ████████████████
    192.168.1.2:8080      3333 ( 33.3%) ████████████████
    192.168.1.3:8080      3333 ( 33.3%) ████████████████

  ── Weighted Round Robin (加权轮询) ──
    192.168.1.1:8080      5000 ( 50.0%) █████████████████████████
    192.168.1.2:8080      3000 ( 30.0%) ███████████████
    192.168.1.3:8080      2000 ( 20.0%) ██████████

  ── Random (随机) ──
    192.168.1.1:8080      3283 ( 32.8%) ████████████████
    192.168.1.2:8080      3358 ( 33.6%) ████████████████
    192.168.1.3:8080      3359 ( 33.6%) ████████████████

  ── Weighted Random (加权随机) ──
    192.168.1.1:8080      4939 ( 49.4%) ████████████████████████
    192.168.1.2:8080      3003 ( 30.0%) ███████████████
    192.168.1.3:8080      2058 ( 20.6%) ██████████

  ── Consistent Hash (一致性哈希) ──
    192.168.1.1:8080      3320 ( 33.2%) ████████████████
    192.168.1.2:8080      3446 ( 34.5%) █████████████████
    192.168.1.3:8080      3234 ( 32.3%) ████████████████

  ★ 平滑加权轮询前 10 次选择（weight=5,3,2）:
    S1 S2 S3 S1 S1 S2 S1 S3 S2 S1 
    → 分布均匀，不会出现连续选同一台（Nginx 算法优势）

=== 三、算法对比 ===

  ┌──────────────────┬──────────────────┬──────────────────────────┐
  │ 算法             │ 适用场景         │ 优缺点                   │
  ├──────────────────┼──────────────────┼──────────────────────────┤
  │ Round Robin      │ 服务器性能相同   │ ✓简单 ✗不考虑权重       │
  │ Weighted RR ★   │ 服务器性能不同   │ ✓平滑分布 ✓Nginx默认   │
  │ Random           │ 大量请求时       │ ✓简单 ✗小样本不均匀     │
  │ Weighted Random  │ 性能不同+简单    │ ✓考虑权重 ✗不够平滑     │
  │ Consistent Hash  │ 有状态服务/缓存  │ ✓会话保持 ✓扩缩容友好  │
  └──────────────────┴──────────────────┴──────────────────────────┘

=== 四、客户端 LB vs 服务端 LB ===

  ┌──────────────┬──────────────────────┬──────────────────────┐
  │              │ 客户端 LB (Ribbon)   │ 服务端 LB (Nginx)    │
  ├──────────────┼──────────────────────┼──────────────────────┤
  │ LB 位置      │ 调用方进程内         │ 独立的 LB 服务器     │
  │ 实例列表     │ 从注册中心拉取       │ 配置文件 upstream    │
  │ 单点故障     │ ✗ 无单点             │ ✓ LB 本身可能故障   │
  │ 性能         │ ✓ 少一跳             │ ✗ 多一跳转发        │
  │ 灵活性       │ ✓ 可定制算法         │ ✗ 算法有限          │
  │ 典型代表     │ Ribbon / LoadBalancer│ Nginx / F5 / HAProxy│
  └──────────────┴──────────────────────┴──────────────────────┘

  ★ 面试必背：
    Q: Ribbon 的负载均衡策略有哪些？
    A: RoundRobinRule（默认）、RandomRule、WeightedResponseTimeRule、
       BestAvailableRule、RetryRule、ZoneAvoidanceRule
