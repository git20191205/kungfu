【重点观察 1】平滑加权轮询算法（Nginx 核心算法）
  ★ 前 10 次选择: S1 S2 S3 S1 S1 S2 S1 S3 S2 S1
  ├─ 不会出现 S1 S1 S1 S1 S1 连续选同一台
  ├─ 算法: currentWeight += weight → 选最大 → 减 totalWeight
  └─ Nginx upstream 默认使用此算法，面试高频考点

【重点观察 2】随机 vs 轮询的精度对比
  ★ 10000 次请求下:
  ├─ 轮询: 偏差 0%（确定性算法）
  ├─ 随机: 偏差 ~0.8%（概率性算法）
  ├─ 加权轮询: 偏差 0%（确定性）
  └─ 加权随机: 偏差 ~0.6%（概率性）
  结论: 小流量场景用轮询更稳定，大流量场景随机也可接受

【重点观察 3】一致性哈希的应用场景
  ★ 同一用户/同一 key → 始终路由到同一节点
  ├─ 适用: 有状态服务、本地缓存、Session 保持
  ├─ 虚拟节点: 解决数据倾斜问题（通常 150-200 个）
  └─ Redis Cluster 分片也用一致性哈希（16384 槽位）

【结论】
  ✅ 加权轮询(WRR) 是生产首选 — 精确 + 平滑 + Nginx 默认
  ✅ 一致性哈希适合有状态场景（缓存/Session）
  ★ 面试关键词: 平滑加权轮询 / 虚拟节点 / 客户端LB vs 服务端LB
  ★ Ribbon 默认 RoundRobinRule，Nginx 默认加权轮询
