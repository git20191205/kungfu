【重点观察 1】JDK 动态代理核心机制
  ★ Proxy.newProxyInstance(classLoader, interfaces, handler)
  ├─ InvocationHandler.invoke() 拦截所有方法调用
  ├─ 解析方法注解 → 构建 RequestTemplate → 发送 HTTP
  └─ 与 Spring AOP 的 JdkDynamicAopProxy 同源（D15）

【重点观察 2】注解解析 → URL 构建
  ★ @GetMapping("/api/orders/{id}") + @PathVariable
  ├─ {id} 占位符替换为实际参数值 "ORD-1001"
  ├─ @RequestParam → 拼接 ?key=value 到 URL
  └─ Content-Type: application/json 默认添加

【重点观察 3】拦截器链设计
  ★ RequestInterceptor 接口 — 请求发送前的切面
  ├─ AuthInterceptor: 注入 Authorization Header（Token 传递）
  ├─ TraceInterceptor: 注入 X-Request-Id（链路追踪）
  └─ 生产中常用: 签名校验、灰度标记、租户ID 透传

【结论】
  ✅ Feign 本质 = JDK 动态代理 + 注解解析 + HTTP Client
  ✅ 声明式调用: 像调本地方法一样调远程服务
  ★ 面试关键词: 动态代理 / RequestTemplate / 拦截器链
  ★ 与 RestTemplate 对比: Feign 代码量少、可读性好、内置 Ribbon 负载均衡
