## 二、核心项目

### 2.1 秒杀系统（w09-seckill）⭐⭐⭐

**项目简介**：高并发秒杀系统，支持 QPS 6711

**技术栈**：
- Spring Boot + Spring Cloud
- MySQL + Redis + Kafka
- Nacos + Sentinel + Gateway
- Docker + Prometheus + Grafana

**核心功能**：
- 秒杀下单（Redis Lua 原子扣库存）
- 异步削峰（Kafka 消息队列）
- 限流熔断（Sentinel）
- 监控告警（Prometheus + Grafana）

**性能数据**：
- QPS：6711（100 并发）
- P99 延迟：28ms
- 成功率：100%
- 无超卖

**技术亮点**：
1. **四层过滤架构**：CDN → Gateway → Redis → Kafka
2. **三道防线防超卖**：Redis Lua + MySQL 乐观锁 + 唯一索引
3. **异步削峰**：Kafka 异步创建订单
4. **完整监控**：Prometheus + Grafana 实时监控

**优化路线**：
- 短期（1周）：JVM 调优 → QPS 8000
- 中期（1月）：分机部署 → QPS 15000
- 长期（3月）：K8s 集群 → QPS 30000+

## 五、技术亮点

### 5.1 秒杀系统性能优化

**优化前**：
- QPS：3623
- P99 延迟：45ms

**优化后**：
- QPS：6711（+85%）
- P99 延迟：28ms（-38%）

**优化手段**：
1. Redis Lua 原子扣库存
2. Kafka 异步削峰
3. 布隆过滤器防止重复购买
4. 连接池预热
5. JVM 参数调优

### 5.2 系统设计能力

**需求分析**：
- 使用 5W1H 方法澄清需求
- SMART 原则拆解需求
- MoSCoW 优先级排序

**容量估算**：
- QPS 估算：DAU × 请求数 / 86400 × 峰值系数
- 存储估算：单条记录 × 记录数 × 副本数
- 服务器估算：峰值 QPS / 单机 QPS × 冗余系数

**架构设计**：
- 四层过滤架构
- 三道防线防超卖
- 异步削峰
- 限流熔断降级

### 5.3 问题排查能力

**CPU 100% 排查**：
```bash
jps → top -Hp <pid> → jstack <pid> → 分析线程栈
```

**内存泄漏排查**：
```bash
jmap -dump → MAT 分析 → 找到泄漏对象 → 分析引用链
```

**慢查询优化**：
