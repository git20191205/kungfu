=== 案例1: 无索引 → 添加索引 ===

  ★ 无索引时：
    SQL: SELECT * FROM t_order WHERE user_id = 42
    id                  select_type         table               partitions          type                possible_keys       key                 key_len             ref                 rows                filtered            Extra               
    1                   SIMPLE              t_order             NULL                ALL                 NULL                NULL                NULL                NULL                49993               10.0                Using where         
    (1 rows)

  添加索引: ALTER TABLE t_order ADD INDEX idx_user_id(user_id)

  ★ 有索引后：
    SQL: SELECT * FROM t_order WHERE user_id = 42
    id                  select_type         table               partitions          type                possible_keys       key                 key_len             ref                 rows                filtered            Extra               
    1                   SIMPLE              t_order             NULL                ref                 idx_user_id         idx_user_id         8                   const               50                  100.0               NULL                
    (1 rows)

  对比要点：
    - type: ALL(全表扫描) → ref(索引查找)
    - key: NULL → idx_user_id
    - rows: ~50000 → ~50（扫描行数大幅减少）

=== 案例2: 联合索引最左前缀 ===

  创建联合索引: (user_id, status, product_id)

  ✓ 命中索引（从 user_id 开始）：
    SQL: SELECT * FROM t_order WHERE user_id = 42 AND status = 1
    id                  select_type         table               partitions          type                possible_keys       key                 key_len             ref                 rows                filtered            Extra               
    1                   SIMPLE              t_order             NULL                ref                 idx_composite       idx_composite       9                   const,const         16                  100.0               NULL                
    (1 rows)

  ✓ 命中索引（只用 user_id）：
    SQL: SELECT * FROM t_order WHERE user_id = 42
    id                  select_type         table               partitions          type                possible_keys       key                 key_len             ref                 rows                filtered            Extra               
    1                   SIMPLE              t_order             NULL                ref                 idx_composite       idx_composite       8                   const               50                  100.0               NULL                
    (1 rows)

  ✗ 不命中（跳过 user_id，只用 status）：
    SQL: SELECT * FROM t_order WHERE status = 1
    id                  select_type         table               partitions          type                possible_keys       key                 key_len             ref                 rows                filtered            Extra               
    1                   SIMPLE              t_order             NULL                ALL                 NULL                NULL                NULL                NULL                49993               10.0                Using where         
    (1 rows)

  ✗ 不命中（跳过 user_id，只用 product_id）：
    SQL: SELECT * FROM t_order WHERE product_id = 100
    id                  select_type         table               partitions          type                possible_keys       key                 key_len             ref                 rows                filtered            Extra               
    1                   SIMPLE              t_order             NULL                ALL                 NULL                NULL                NULL                NULL                49993               10.0                Using where         
    (1 rows)

  最左前缀规则：
    索引 (a, b, c) 能命中的查询：
    ✓ WHERE a=1
    ✓ WHERE a=1 AND b=2
    ✓ WHERE a=1 AND b=2 AND c=3
    ✗ WHERE b=2（跳过了 a）
    ✗ WHERE c=3（跳过了 a, b）
    ✗ WHERE b=2 AND c=3（跳过了 a）

=== 案例3: 回表 → 覆盖索引 ===

  ★ 回表查询（SELECT * 需要回聚簇索引）：
    SQL: SELECT * FROM t_order WHERE user_id = 42 AND status = 1
    id                  select_type         table               partitions          type                possible_keys       key                 key_len             ref                 rows                filtered            Extra               
    1                   SIMPLE              t_order             NULL                ref                 idx_composite       idx_composite       9                   const,const         16                  100.0               NULL                
    (1 rows)

  ★ 覆盖索引（只查 user_id, status, product_id — 都在索引里）：
    SQL: SELECT user_id, status, product_id FROM t_order WHERE user_id = 42 AND status = 1
    id                  select_type         table               partitions          type                possible_keys       key                 key_len             ref                 rows                filtered            Extra               
    1                   SIMPLE              t_order             NULL                ref                 idx_composite       idx_composite       9                   const,const         16                  100.0               Using index         
    (1 rows)

  对比要点：
    - 覆盖索引的 Extra 列出现 'Using index'
    - 说明只读索引就够了，不需要回表读数据行
    - 回表查询的 Extra 没有 'Using index'，需要额外 IO

  ★ 实战建议：
    1. 高频查询尽量只 SELECT 需要的列，而非 SELECT *
    2. 把高频查询的列加入联合索引，实现覆盖索引
    3. 索引不是越多越好 — 每个索引都有写入和存储开销
