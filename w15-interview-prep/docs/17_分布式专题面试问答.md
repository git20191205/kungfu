# 分布式专题 — 高频面试问答

> 基于 w05-w07 和 w09-seckill 项目经验整理的 20 个高频问题

## 一、分布式理论（5 题）

### 1. 什么是 CAP 理论？

**答**：
- **C（Consistency）**：一致性，所有节点数据一致
- **A（Availability）**：可用性，系统一直可用
- **P（Partition Tolerance）**：分区容错性，网络分区时仍能工作

**定理**：最多同时满足两个
- **CP**：ZooKeeper（牺牲可用性）
- **AP**：Cassandra（牺牲一致性）

### 2. 什么是 BASE 理论？

**答**：
- **BA（Basically Available）**：基本可用
- **S（Soft State）**：软状态，允许中间状态
- **E（Eventually Consistent）**：最终一致性

### 3. 什么是分布式事务？如何实现？

**答**：
- **2PC**：两阶段提交，强一致性，性能差
- **TCC**：Try-Confirm-Cancel，最终一致性
- **Saga**：长事务，补偿机制
- **本地消息表**：最终一致性
- **MQ 事务消息**：RocketMQ

### 4. 什么是分布式 ID？如何生成？

**答**：
- **UUID**：简单，无序，36 位
- **Snowflake**：64 位，有序，高性能（推荐）
- **数据库自增**：简单，性能差
- **Redis INCR**：简单，依赖 Redis

### 5. 什么是分布式锁？如何实现？

**答**：
- **Redis**：SETNX + 过期时间
- **ZooKeeper**：临时顺序节点
- **数据库**：唯一索引

## 二、微服务（5 题）

### 6. 微服务的优缺点？

**答**：
**优点**：
- 独立开发、部署、扩展
- 技术栈多样化
- 故障隔离

**缺点**：
- 分布式复杂性
- 数据���致性
- 运维成本高

### 7. 服务注册与发现的原理？

**答**：
- **服务注册**：服务启动时注册到注册中心（Nacos / Eureka）
- **服务发现**：客户端从注册中心获取服务列表
- **健康检查**：定期心跳，剔除不健康服务

### 8. 负载均衡的策略有哪些？

**答**：
- **轮询（Round Robin）**：依次分配
- **随机（Random）**：随机选择
- **加权轮询**：按权重分配
- **最少连接**：选择连接数最少的
- **一致性 Hash**：相同请求路由到同一节点

### 9. 服务限流的算法有哪些？

**答**：
- **固定窗口**：简单，有突刺问题
- **滑动窗口**：平滑，复杂
- **令牌桶**：平滑，允许突发（推荐）
- **漏桶**：平滑，不允许突发

### 10. 服务熔断的原理？

**答**：
- **关闭（Closed）**：正常调用
- **打开（Open）**：失败率 > 50%，拒绝所有请求
- **半开（Half-Open）**：等待 30 秒，允许少量请求
- **恢复**：成功率 > 80%，关闭熔断

## 三、消息队列（5 题）

### 11. 消息队列的作用？

**答**：
- **异步处理**：提升响应速度
- **削峰填谷**：保护下游系统
- **解耦**：服务之间解耦

### 12. Kafka 的架构？

**答**：
- **Broker**：Kafka 服务器
- **Topic**：消息主题
- **Partition**：分区，并行消费
- **Producer**：生产者
- **Consumer**：消费者
- **Consumer Group**：消费者组

### 13. 如何保证消息不丢失？

**答**：
- **生产者**：acks=all，等待所有副本确认
- **Broker**：副本机制，min.insync.replicas=2
- **消费者**：手动提交 offset

### 14. 如何保证消息不重复？

**答**：
- **生产者**：幂等性，enable.idempotence=true
- **消费者**：幂等处理（唯一索引、Redis SETNX）

### 15. 如何保证消息顺序？

**答**：
- **单 Partition**：同一 Key 路由到同一 Partition
- **单消费者**：一个 Partition 只能被一个消费者消费

## 四、分布式缓存（5 题）

### 16. Redis Cluster 的原理？

**答**：
- **Hash Slot**：16384 个 slot，CRC16(key) % 16384
- **主从复制**：每个 Master 有 Slave
- **故障转移**：Master 故障，Slave 自动提升

### 17. 一致性 Hash 的原理？

**答**：
- **Hash 环**：0 ~ 2^32-1
- **节点映射**：hash(node) 映射到环上
- **数据映射**：hash(key) 顺时针找到第一个节点
- **虚拟节点**：解决数据分布不均

### 18. 缓存更新策略有哪些？

**答**：
- **Cache Aside**：先更新数据库，再删除缓存（推荐）
- **Read/Write Through**：缓存代理数据库
- **Write Behind**：异步写入数据库

### 19. 如何设计缓存 Key？

**答**：
```
业务模块:对象类型:ID:属性

示例：
seckill:activity:1           # 活动信息
seckill:stock:1              # 库存
seckill:order:10001:status   # 订单状态
```
