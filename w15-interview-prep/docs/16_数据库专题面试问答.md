# 数据库专题 — 高频面试问答

> 基于 w04-mysql-redis 和 w09-seckill 项目经验整理的 20 个高频问题

## 一、MySQL 基础（5 题）

### 1. MySQL 的存储引擎有哪些？

**答**：
- **InnoDB**：支持事务、行锁、外键（默认）
- **MyISAM**：不支持事务、表锁、全文索引
- **Memory**：内存存储，快速，重启丢失

### 2. 索引的类型有哪些？

**答**：
- **主键索引**：PRIMARY KEY，唯一且非空
- **唯一索引**：UNIQUE，唯一但可为空
- **普通索引**：INDEX，加速查询
- **全文索引**：FULLTEXT，文本搜索
- **联合索引**：多列组合

### 3. 什么是聚簇索引和非聚簇索引？

**答**：
- **聚簇索引**：数据和索引存在一起，InnoDB 主键索引
- **非聚簇索引**：索引和数据分开，MyISAM 所有索引

### 4. 索引的数据结构是什么？

**答**：
- **B+Tree**：多路平衡树，叶子节点存数据，范围查询快
- **Hash**：等值查询快，不支持范围查询

### 5. 什么是覆盖索引？

**答**：
- **定义**：查询的列都在索引中，不需要回表
- **优点**：减少 IO，提升性能

```sql
-- 联合索引
CREATE INDEX idx_name_age ON user(name, age);

-- 覆盖索引查询
SELECT name, age FROM user WHERE name = 'Alice';  -- 不需要回表
```

## 二、SQL 优化（5 题）

### 6. 如何优化慢查询？

**答**：
1. **开启慢查询日志**：`slow_query_log = 1`
2. **分析执行计划**：`EXPLAIN SELECT ...`
3. **优化索引**：添加索引、优化联合索引
4. **优化 SQL**：避免 SELECT *、避免子查询
5. **分库分表**：数据量大时

### 7. EXPLAIN 的关键字段？

**答**：
- **type**：访问类型（ALL < index < range < ref < eq_ref < const）
- **key**：使用的索引
- **rows**：扫描行数
- **Extra**：额外信息（Using index = 覆盖索引）

### 8. 索引失效的场景？

**答**：
- **不满足最左前缀**：联合索引 (a, b, c)，查询 WHERE b = ?
- **使用函数**：WHERE YEAR(create_time) = 2024
- **类型转换**：WHERE age = '18'（age 是 int）
- **OR 条件**：WHERE a = 1 OR b = 2
- **!=、<>、NOT IN**：无法使用索引

### 9. 如何优化分页查询？

**答**：
```sql
-- 深分页慢（扫描 + 丢弃前 1000000 行）
SELECT * FROM user LIMIT 1000000, 10;

-- 优化 1：子查询（只扫描索引）
SELECT * FROM user WHERE id >= (
    SELECT id FROM user LIMIT 1000000, 1
) LIMIT 10;

-- 优化 2：游标分页
SELECT * FROM user WHERE id > 1000000 LIMIT 10;
```

### 10. 秒杀系统的 SQL 优化经验？

**答**（基于 w09-seckill 项目）：
```sql
-- 1. 唯一索引防止重复购买
CREATE UNIQUE INDEX uk_activity_user ON seckill_order(activity_id, user_id);

-- 2. 联合索引优化查询
CREATE INDEX idx_user_time ON seckill_order(user_id, create_time);

-- 3. 乐观锁防止超卖
UPDATE seckill_activity 
SET remain_stock = remain_stock - 1 
WHERE id = ? AND remain_stock > 0;

-- 效果：QPS 6711，查询 < 10ms
```

## 三、事务与锁（5 题）

### 11. 事务的 ACID 特性？

**答**：
- **原子性（Atomicity）**：全部成功或全部失败
- **一致性（Consistency）**：数据完整性约束
- **隔离性（Isolation）**：事务之间互不干扰
- **持久性（Durability）**：提交后永久保存

### 12. 事务的隔离级别有哪些？

**答**：
- **读未提交（READ UNCOMMITTED）**：脏读、不可重复��、幻读
- **读已提交（READ COMMITTED）**：不可重复读、幻读
- **可重复读（REPEATABLE READ）**：幻读（MySQL 默认）
- **串行化（SERIALIZABLE）**：无并发问题，性能差

### 13. 什么是 MVCC？

**答**：
- **定义**：多版本并发控制，读不加锁
- **原理**：每行记录有隐藏列（事务 ID、回滚指针）
- **快照读**：读取历史版本，不加锁
- **当前读**：读取最新版本，加锁（SELECT FOR UPDATE）

### 14. MySQL 的锁有哪些？

**答**：
- **表锁**：锁整张表，MyISAM
- **行锁**：锁单行，InnoDB
- **间隙锁**：锁范围，防止幻读
- **意向锁**：表级锁，标识行锁意图

### 15. 如何避免死锁？

**答**：
- **按顺序加锁**：统一加锁顺序
- **减少锁持有时间**：尽快提交事务
- **使用乐观锁**：version 字段
- **设置超时**：`innodb_lock_wait_timeout`

## 四、Redis 缓存（5 题）

### 16. Redis 的数据类型有哪些？

**答**：
- **String**：字符串，SET / GET
- **Hash**：哈希表，HSET / HGET
- **List**：列表，LPUSH / RPOP
- **Set**：集合，SADD / SMEMBERS
- **ZSet**：有序集合，ZADD / ZRANGE

### 17. Redis 的持久化方式有哪些？

**答**：
- **RDB**：快照，定期保存，恢复快，可能丢数据
- **AOF**：追加日志，实时保存，恢复慢，数据完整

### 18. 缓存穿透、击穿、雪崩如何解决？

**答**：
- **穿透**：查询不存在的数据，布隆过滤器 / 缓存空值
- **击穿**：热点数据过期，分布式锁 / 永不过期
- **雪崩**：大量数据同时过期，随机过期时间 / 多级缓存

### 19. Redis 如何实现分布式锁？

**答**：
```java
// 1. 加锁
SET lock_key unique_value NX EX 30

// 2. 解锁（Lua 脚本保证原子性）

    return 0
end
```
