# 秒杀系统 — 接口文档

## 基础信息

| 项目 | 说明 |
|------|------|
| 网关地址 | `http://localhost:8090` |
| 数据格式 | JSON |
| 字符编码 | UTF-8 |

### 统一响应格式

```json
{
  "code": 200,
  "msg": "ok",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 200=成功，400=业务错误，404=不存在，429=限流，500=系统错误 |
| msg | String | 提示信息 |
| data | Object | 业务数据（失败时为 null） |

---

## 一、秒杀接口

### 1.1 执行秒杀

```
POST /api/seckill/{activityId}
```

| 参数 | 位置 | 必填 | 说明 |
|------|------|------|------|
| activityId | path | 是 | 秒杀活动 ID |
| X-User-Id | header | 是 | 用户 ID |

成功响应：
```json
{
  "code": 200,
  "msg": "ok",
  "data": "SK_1776685328284_10001"
}
```

`data` 为订单号，用于后续查询秒杀结果。

失败响应：
```json
{ "code": 400, "msg": "活动未开始", "data": null }
{ "code": 400, "msg": "已售罄", "data": null }
{ "code": 400, "msg": "您已购买过", "data": null }
{ "code": 400, "msg": "请求过于频繁，请稍后再试", "data": null }
```

处理流程：
1. 用户频率限制（Redis SETNX，5 秒内只能请求 1 次）
2. 布隆过滤器检查是否已购买
3. Redis Lua 原子扣减库存
4. 发送 Kafka 消息到订单服务
5. 返回订单号

---

### 1.2 查询秒杀结果

```
GET /api/seckill/result/{orderNo}
```

| 参数 | 位置 | 必填 | 说明 |
|------|------|------|------|
| orderNo | path | 是 | 订单号（秒杀接口返回） |

成功响应：
```json
{
  "code": 200,
  "msg": "ok",
  "data": {
    "orderNo": "SK_1776685328284_10001",
    "status": "PAID"
  }
}
```

| status 值 | 说明 |
|-----------|------|
| PENDING | 排队中 / 待支付 |
| PAID | 已支付 |
| CANCELLED | 已取消 |
| TIMEOUT | 超时未支付 |

## 二、活动接口

### 2.1 查询活动详情

```
GET /api/seckill/activity/{activityId}
```

| 参数 | 位置 | 必填 | 说明 |
|------|------|------|------|
| activityId | path | 是 | 活动 ID |

成功响应：
```json
{
  "code": 200,
  "msg": "ok",
  "data": {
    "activityName": "iPhone 15 限时秒杀",
    "productName": "iPhone 15 128GB",
    "originalPrice": 5999.00,
    "seckillPrice": 4999.00,
    "remainStock": 42,
    "startTime": "2026-04-21T10:00:00",
    "endTime": "2026-04-21T10:30:00",
    "status": "ACTIVE"
  }
}
```

| status 值 | 说明 |
|-----------|------|
| PENDING | 未开始 |
| ACTIVE | 进行中 |
| ENDED | 已结束 |

---

### 2.2 活动预热

```
POST /api/seckill/activity/{activityId}/warmup
```

| 参数 | 位置 | 必填 | 说明 |
|------|------|------|------|
| activityId | path | 是 | 活动 ID |

成功响应：
```json
{
  "code": 200,
  "msg": "ok",
  "data": "预热完成"
}
```

说明：秒杀开始前调用，将活动信息和库存加载到 Redis 缓存。

## 三、支付接口

### 3.1 查询支付状态

```
GET /api/seckill/payment/{paymentNo}
```

| 参数 | 位置 | 必填 | 说明 |
|------|------|------|------|
| paymentNo | path | 是 | 支付单号 |

成功响应：
```json
{
  "code": 200,
  "msg": "ok",
  "data": {
    "id": 1,
    "paymentNo": "PAY_1776685330000_12345",
    "orderNo": "SK_1776685328284_10001",
    "amount": 4999.00,
    "status": "PAID",
    "createTime": "2026-04-21T10:00:02",
    "updateTime": "2026-04-21T10:00:04"
  }
}
```

| status 值 | 说明 |
|-----------|------|
| PENDING | 待支付 |
| PAID | 已支付 |
| FAILED | 支付失败 |

---

### 3.2 手动触发支付回调（测试用）

```
POST /api/seckill/payment/callback/{paymentNo}?success=true
```

| 参数 | 位置 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| paymentNo | path | 是 | - | 支付单号 |
| success | query | 否 | true | 是否支付成功 |

成功响应：
```json
{ "code": 200, "msg": "ok", "data": null }
```

## 四、服务端口

| 服务 | 端口 | Nacos 服务名 |
|------|------|-------------|
| Gateway | 8090 | seckill-gateway |
| Seckill Service | 8084 | seckill-service |
| Order Service | 8085 | seckill-order |
| Payment Service | 8086 | seckill-payment |

## 五、Kafka Topic

| Topic | 生产者 | 消费者 | 说明 |
|-------|--------|--------|------|
| seckill-order | seckill-service | seckill-order | 秒杀成功后异步创建订单 |
| seckill-payment | seckill-order | seckill-payment | 订单创建后发起支付 |
| payment-result | seckill-payment | seckill-order | 支付结果回调更新订单状态 |
