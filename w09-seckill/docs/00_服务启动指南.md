# W09 秒杀系统 — 服务启动指南

## 前提条件

**必须按顺序确认以下中间件已启动并就绪：**

### 1. MySQL (3306)
```bash
mysql -u root -p123456 -e "SELECT 1"
```
预期：返回结果无错误

### 2. Redis (6379)
```bash
redis-cli ping
```
预期：返回 `PONG`

### 3. Kafka (9092)
```bash
# Windows 检查端口
netstat -an | findstr "9092"
```
预期：看到 `LISTENING`

### 4. Nacos (8848) ⚠️ 重要
```bash
# 启动 Nacos（如果未启动）
cd D:\nacos\bin
startup.cmd -m standalone

# 等待 30-60 秒，然后验证
curl http://localhost:8848/nacos/v1/console/health/readiness
```
预期：返回 `UP` 或 HTTP 200

**⚠️ 关键：必须等待 Nacos 完全启动（约 30-60 秒）后再启动微服务！**

浏览器访问 http://localhost:8848/nacos 能看到登录页面，说明 Nacos 已就绪。

## 一、数据库初始化

### 1. 创建数据库和表

```sql
-- 连接 MySQL
mysql -u root -p123456

-- 创建数据库
CREATE DATABASE IF NOT EXISTS kungfu_seckill DEFAULT CHARACTER SET utf8mb4;
USE kungfu_seckill;

-- 秒杀活动表
CREATE TABLE t_seckill_activity (
    id             BIGINT AUTO_INCREMENT PRIMARY KEY,
    activity_name  VARCHAR(100) NOT NULL COMMENT '活动名称',
    product_id     BIGINT NOT NULL COMMENT '商品ID',
    product_name   VARCHAR(100) COMMENT '商品名称',
    original_price DECIMAL(10,2) NOT NULL COMMENT '原价',
    seckill_price  DECIMAL(10,2) NOT NULL COMMENT '秒杀价',
    total_stock    INT NOT NULL COMMENT '总库存',
    start_time     DATETIME NOT NULL COMMENT '开始时间',
    end_time       DATETIME NOT NULL COMMENT '结束时间',
    status         VARCHAR(20) NOT NULL DEFAULT 'PENDING' COMMENT 'PENDING/ACTIVE/ENDED',
    create_time    DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_status_time (status, start_time)
) ENGINE=InnoDB COMMENT='秒杀活动表';

-- 秒杀库存表（独立，减少锁竞争）
CREATE TABLE t_seckill_stock (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    activity_id BIGINT NOT NULL UNIQUE COMMENT '活动ID',
    quantity    INT NOT NULL DEFAULT 0 COMMENT '可用库存',
    INDEX idx_activity (activity_id)
) ENGINE=InnoDB COMMENT='秒杀库存表';

-- 秒杀订单表
CREATE TABLE t_seckill_order (
    id            BIGINT AUTO_INCREMENT PRIMARY KEY,
    order_no      VARCHAR(64) NOT NULL UNIQUE COMMENT '订单号',
    activity_id   BIGINT NOT NULL COMMENT '活动ID',
    user_id       BIGINT NOT NULL COMMENT '用户ID',
    product_id    BIGINT NOT NULL COMMENT '商品ID',
    seckill_price DECIMAL(10,2) NOT NULL COMMENT '秒杀价',
    quantity      INT NOT NULL DEFAULT 1,
    status        VARCHAR(20) NOT NULL DEFAULT 'PENDING' COMMENT 'PENDING/PAID/CANCELLED/TIMEOUT',
    create_time   DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time   DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_activity_user (activity_id, user_id) COMMENT '同一活动同一用户只能买一次'
) ENGINE=InnoDB COMMENT='秒杀订单表';

-- Seata undo_log（如果使用 Seata）
CREATE TABLE IF NOT EXISTS undo_log (
    branch_id     BIGINT NOT NULL,
    xid           VARCHAR(128) NOT NULL,
    context       VARCHAR(128) NOT NULL,
    rollback_info LONGBLOB NOT NULL,
    log_status    INT NOT NULL,
    log_created   DATETIME(6) NOT NULL,
    log_modified  DATETIME(6) NOT NULL,
    UNIQUE KEY ux_undo_log (xid, branch_id)
) ENGINE=InnoDB;

-- 初始化测试数据：一场秒杀活动
INSERT INTO t_seckill_activity
  (activity_name, product_id, product_name, original_price, seckill_price, total_stock, start_time, end_time, status)
VALUES
  ('iPhone 15 限时秒杀', 1, 'iPhone 15', 5999.00, 3999.00, 100,
   '2026-04-21 10:00:00', '2026-04-30 23:59:59', 'ACTIVE');

INSERT INTO t_seckill_stock (activity_id, quantity) VALUES (1, 100);
```

### 2. 验证数据

```sql
-- 查看活动
SELECT * FROM t_seckill_activity;

-- 查看库存
SELECT * FROM t_seckill_stock;
```

### 方式3: 使用优化后的 JVM 参数启动（推荐用于性能测试）

#### Gateway (256MB 堆内存)
```bash
cd D:\workspace\kungfu\w09-seckill\seckill-gateway\target
java -Xms256m -Xmx256m -XX:+UseG1GC -XX:MaxGCPauseMillis=50 ^
  -XX:+HeapDumpOnOutOfMemoryError -XX:HeapDumpPath=./heapdump.hprof ^
  -jar seckill-gateway-1.0-SNAPSHOT.jar
```

#### Seckill Service (512MB 堆内存)
```bash
cd D:\workspace\kungfu\w09-seckill\seckill-service\target
java -Xms512m -Xmx512m -XX:+UseG1GC -XX:MaxGCPauseMillis=50 ^
  -XX:+HeapDumpOnOutOfMemoryError -XX:HeapDumpPath=./heapdump.hprof ^
  -jar seckill-service-1.0-SNAPSHOT.jar
```

#### Order Service (512MB 堆内存)
```bash
cd D:\workspace\kungfu\w09-seckill\seckill-order\target
java -Xms512m -Xmx512m -XX:+UseG1GC -XX:MaxGCPauseMillis=50 ^
  -XX:+HeapDumpOnOutOfMemoryError -XX:HeapDumpPath=./heapdump.hprof ^
  -jar seckill-order-1.0-SNAPSHOT.jar
```

#### Payment Service (256MB 堆内存)
```bash
cd D:\workspace\kungfu\w09-seckill\seckill-payment\target
java -Xms256m -Xmx256m -XX:+UseG1GC -XX:MaxGCPauseMillis=50 ^
  -XX:+HeapDumpOnOutOfMemoryError -XX:HeapDumpPath=./heapdump.hprof ^
  -jar seckill-payment-1.0-SNAPSHOT.jar
```

**JVM 参数说明**：
- `-Xms` / `-Xmx`: 固定堆内存大小，避免动态扩容
- `-XX:+UseG1GC`: 使用 G1 垃圾收集器
- `-XX:MaxGCPauseMillis=50`: 目标 GC 停顿时间 50ms
- `-XX:+HeapDumpOnOutOfMemoryError`: OOM 时自动生成 heap dump

## 六、常见问题

### 1. 服务启动失败

**检查日志**：查看控制台输出的错误信息

**常见原因**：
- Nacos 未启动或连接失败
- MySQL/Redis/Kafka 未启动
- 端口被占用

### 2. Nacos 注册失败

**检查配置**：`application.yml` 中的 Nacos 地址是否正确
```yaml
spring:
  cloud:
    nacos:
      discovery:
        server-addr: localhost:8848
```

### 3. Redis 连接失败

**检查 Redis**：
```bash
redis-cli ping
```

**检查配置**：
```yaml
spring:
  redis:
    host: localhost
    port: 6379
```

### 4. Kafka 连接失败

**检查 Kafka**：
```bash
# Windows
netstat -an | findstr "9092"

# 查看 topic
kafka-topics.bat --list --bootstrap-server localhost:9092
```
