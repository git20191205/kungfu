========================================
  JVM 内存区域全景演示
========================================

【1. 堆 Heap】— 存放对象实例
  [堆] 创建对象 demo，实例字段: 我是实例字段，跟随对象分配在堆上
  [堆] 数组对象也在堆上，长度: 3

【2. 虚拟机栈 VM Stack】— 存放栈帧（局部变量表、操作数栈等）
  [虚拟机栈] 第 1 层栈帧，局部变量 localVar 存在栈帧中
  [虚拟机栈] 第 2 层栈帧，局部变量 localVar 存在栈帧中
  [虚拟机栈] 第 3 层栈帧，局部变量 localVar 存在栈帧中

【3. 方法区 Method Area】— 存放类信息、常量、静态变量
  [方法区] 静态常量 CONSTANT = 我是常量，存在方法区的运行时常量池
  [方法区] 静态变量 staticCounter = 1
  [方法区] 类信息: com.kungfu.jvm.d1_memory_model.MemoryAreaDemo

【4. 本地方法栈 Native Method Stack】— 为 native 方法服务
  [本地方法栈] System.currentTimeMillis() 是 native 方法，返回: 1773924142537
  [本地方法栈] Object.hashCode() 也是 native 方法，返回: 1603195447

【5. 程序计数器 PC Register】— 记录当前线程执行的字节码行号
  [PC] 程序计数器是线程私有的，无法直接打印
  [PC] 每个线程都有独立的 PC，用于线程切换后恢复执行位置
  [PC] 执行 Java 方法时记录字节码地址，执行 native 方法时为空（Undefined）

========================================
  内存区域速记：
  线程私有：虚拟机栈、本地方法栈、程序计数器
  线程共享：堆、方法区
  堆是 GC 的主战场，方法区回收的是类卸载
