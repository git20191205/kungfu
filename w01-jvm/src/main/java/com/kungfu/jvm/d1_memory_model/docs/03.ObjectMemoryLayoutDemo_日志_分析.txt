● 逐段详细解释

  一、对象头 — Mark Word（第 9-26 行）

  32 位 JVM → 4 bytes
  64 位 JVM → 8 bytes

  ┌─────────────────┬──────────────────────────────────────────────────┐
  │       值        │                       含义                       │
  ├─────────────────┼──────────────────────────────────────────────────┤
  │ 32 位 → 4 bytes │ 32 bit = 4 字节。32 位 JVM 已经很少用了          │
  ├─────────────────┼──────────────────────────────────────────────────┤
  │ 64 位 → 8 bytes │ 你的环境是 64 位 JDK 8，所以 Mark Word = 8 bytes │
  └─────────────────┴──────────────────────────────────────────────────┘

  Mark Word 五种状态（第 14-22 行）：

  Mark Word 的 64 bit 在不同锁状态下存储不同内容（复用同一块空间）：

  ┌──────────┬────────────────────────────────────────────────────────────────────────────┬────────────────────────────────────────────────┐
  │   状态   │                                  存储内容                                  │                    何时进入                    │
  ├──────────┼────────────────────────────────────────────────────────────────────────────┼────────────────────────────────────────────────┤
  │ 无锁     │ hashCode(31bit) + GC年龄(4bit) + 偏向标志(1bit) + 锁标志(2bit)             │ 对象刚创建，没调过 hashCode 时 hashCode 位为 0 │
  ├──────────┼────────────────────────────────────────────────────────────────────────────┼────────────────────────────────────────────────┤
  │ 偏向锁   │ 线程ID(54bit) + epoch(2bit) + GC年龄(4bit) + 偏向标志(1bit) + 锁标志(2bit) │ 第一个线程进入 synchronized                    │
  ├──────────┼────────────────────────────────────────────────────────────────────────────┼────────────────────────────────────────────────┤
  │ 轻量级锁 │ 指向栈中 Lock Record 的指针(62bit) + 锁标志(2bit)                          │ 有竞争但不激烈，CAS 自旋获取锁                 │
  ├──────────┼────────────────────────────────────────────────────────────────────────────┼────────────────────────────────────────────────┤
  │ 重量级锁 │ 指向 Monitor 对象的指针(62bit) + 锁标志(2bit)                              │ 竞争激烈，膨胀为操作系统互斥锁                 │
  ├──────────┼────────────────────────────────────────────────────────────────────────────┼────────────────────────────────────────────────┤
  │ GC 标记  │ 空 + 锁标志(2bit)=11                                                       │ GC 标记阶段，标记对象是否可回收                │
  └──────────┴────────────────────────────────────────────────────────────────────────────┴────────────────────────────────────────────────┘

  三个关键推论（第 24-26 行）：

  ┌────────────────────────────────────────┬────────────────────────────────────────────────────────────────────────────────────────────────────────┐
  │                  输出                  │                                                  含义                                                  │
  ├────────────────────────────────────────┼────────────────────────────────────────────────────────────────────────────────────────────────────────┤
  │ hashCode 在无锁状态时存在 Mark Word 中 │ 调用 hashCode() 后值写入 Mark Word。注意：如果没调过 hashCode()，那 31 bit 全是 0                      │
  ├────────────────────────────────────────┼────────────────────────────────────────────────────────────────────────────────────────────────────────┤
  │ 加了 synchronized 后，hashCode 被挤走  │ 偏向锁需要存线程 ID，没地方放 hashCode → 如果对象已经算过 hashCode，就无法进入偏向锁状态，直接轻量级锁 │
  ├────────────────────────────────────────┼────────────────────────────────────────────────────────────────────────────────────────────────────────┤
  │ GC 年龄只有 4 bit → 最大值 15          │ 4 bit 能表示 0~15，所以 -XX:MaxTenuringThreshold 设超过 15 无效                                        │
  └────────────────────────────────────────┴────────────────────────────────────────────────────────────────────────────────────────────────────────┘

  一、对象头 — 类型指针（第 28-31 行）

  开启压缩指针（-XX:+UseCompressedOops，默认开启）→ 4 bytes
  关闭压缩指针 → 8 bytes

  ┌───────────────────┬─────────────────────────────────────────────────────────────────┐
  │        值         │                              含义                               │
  ├───────────────────┼─────────────────────────────────────────────────────────────────┤
  │ UseCompressedOops │ Compressed Ordinary Object Pointers — 压缩普通对象指针          │
  ├───────────────────┼─────────────────────────────────────────────────────────────────┤
  │ 默认开启          │ 堆 < 32GB 时自动开启（你的环境是开启的）                        │
  ├───────────────────┼─────────────────────────────────────────────────────────────────┤
  │ 4 bytes           │ 压缩后指针从 8 字节缩到 4 字节，每个对象省 4 字节，堆大时很可观 │
  ├───────────────────┼─────────────────────────────────────────────────────────────────┤
  │ 关闭 → 8 bytes    │ 堆 > 32GB 时自动关闭，或手动 -XX:-UseCompressedOops             │
  └───────────────────┴─────────────────────────────────────────────────────────────────┘

  Klass Pointer 指向什么？ 方法区（Metaspace）中该类的 Klass 结构体，包含类名、方法表、字段表等。JVM 通过这个指针知道对象是什么类型。

  一、对象头 — 数组长度（第 33-34 行）

  4 bytes，记录数组的 length

  只有数组对象才有这个。普通对象没有。arr.length 就是读这 4 个字节。4 bytes = 32 bit → 数组最大长度 = 2^31 - 1 = Integer.MAX_VALUE。

  二、实例数据（第 38-51 行）

  boolean 1 byte, byte 1 byte, short 2, char 2, int 4, float 4, long 8, double 8, reference 4

  ┌───────────────┬─────────┬─────────────────────────────────────────────────┐
  │     类型      │  大小   │                      说明                       │
  ├───────────────┼─────────┼─────────────────────────────────────────────────┤
  │ boolean       │ 1 byte  │ 虽然逻辑上只需 1 bit，但 JVM 以 byte 为最小单位 │
  ├───────────────┼─────────┼─────────────────────────────────────────────────┤
  │ byte          │ 1 byte  │ -128 ~ 127                                      │
  ├───────────────┼─────────┼─────────────────────────────────────────────────┤
  │ short / char  │ 2 bytes │ short: -32768~32767, char: 0~65535 (Unicode)    │
  ├───────────────┼─────────┼─────────────────────────────────────────────────┤
  │ int / float   │ 4 bytes │ int: ±21亿, float: IEEE 754 单精度              │
  ├───────────────┼─────────┼─────────────────────────────────────────────────┤
  │ long / double │ 8 bytes │ long: ±9.2×10^18, double: IEEE 754 双精度       │
  ├───────────────┼─────────┼─────────────────────────────────────────────────┤
  │ reference     │ 4 bytes │ 压缩指针开启时对象引用 4 字节，关闭时 8 字节    │
  └───────────────┴─────────┴─────────────────────────────────────────────────┘

  三、对齐填充（第 53-56 行）

  HotSpot 要求对象大小必须是 8 字节的倍数

  ┌──────────────────────┬─────────────────────────────────────────────────────────────────────────────────┐
  │          值          │                                      含义                                       │
  ├──────────────────────┼─────────────────────────────────────────────────────────────────────────────────┤
  │ 8 字节的倍数         │ 所有对象大小必须是 8 的倍数：8, 16, 24, 32, 40...                               │
  ├──────────────────────┼─────────────────────────────────────────────────────────────────────────────────┤
  │ 不够的部分用 0 填充  │ 纯粹占位，没有任何语义                                                          │
  ├──────────────────────┼─────────────────────────────────────────────────────────────────────────────────┤
  │ CPU 访问对齐内存更快 │ 64 位 CPU 一次读 8 字节，对齐后一次读取就能拿到完整数据，不需要跨两次读取再拼接 │
  └──────────────────────┴─────────────────────────────────────────────────────────────────────────────────┘

  四、7 种对象大小计算

  1. new Object()（第 60-69 行）

  Mark Word: 8 + Class Pointer: 4 + 实例数据: 0 = 12 bytes → 补到 16
  hashCode = 1603195447

  ┌───────────────────────┬───────────────────────────────────────────────────────────────────────────────────────────────┐
  │          值           │                                             含义                                              │
  ├───────────────────────┼───────────────────────────────────────────────────────────────────────────────────────────────┤
  │ 实例数据: 0 bytes     │ Object 类没有任何实例字段                                                                     │
  ├───────────────────────┼───────────────────────────────────────────────────────────────────────────────────────────────┤
  │ 小计 12 bytes         │ 不是 8 的倍数                                                                                 │
  ├───────────────────────┼───────────────────────────────────────────────────────────────────────────────────────────────┤
  │ 对齐填充 4 bytes      │ 12 + 4 = 16，满足 8 倍数要求                                                                  │
  ├───────────────────────┼───────────────────────────────────────────────────────────────────────────────────────────────┤
  │ ★ 总计 16 bytes       │ 面试答案：new Object() = 16 字节                                                              │
  ├───────────────────────┼───────────────────────────────────────────────────────────────────────────────────────────────┤
  │ hashCode = 1603195447 │ 调用 hashCode() 后这个值被写入 Mark Word 的高 31 bit 中。每次运行可能不同（基于内存地址计算） │
  └───────────────────────┴───────────────────────────────────────────────────────────────────────────────────────────────┘

  2. new EmptyObject()（第 71-72 行）

  和 Object 一样 = 16 bytes

  空类没有任何字段，和 Object 一样。任何对象最小 16 字节（压缩指针开启时）。

  3. new OneIntObject()（第 74-80 行）

  Mark Word(8) + Class Pointer(4) + int(4) = 16 bytes → 刚好，不用填充

  ┌────────────────────┬───────────────────────────────────────────────────────────────┐
  │         值         │                             含义                              │
  ├────────────────────┼───────────────────────────────────────────────────────────────┤
  │ int value: 4 bytes │ 一个 int 字段                                                 │
  ├────────────────────┼───────────────────────────────────────────────────────────────┤
  │ 小计 16 bytes      │ 16 是 8 的倍数，不需要填充                                    │
  ├────────────────────┼───────────────────────────────────────────────────────────────┤
  │ 和空对象一样大     │ 空对象有 4 bytes 填充，加了一个 int 正好填满填充位 → 大小没变 │
  └────────────────────┴───────────────────────────────────────────────────────────────┘

  4. new OneLongObject()（第 82-87 行）

  Mark Word(8) + Class Pointer(4) + gap(4) + long(8) = 24 bytes

  ┌───────────────────┬───────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┐
  │        值         │                                                                   含义                                                                    │
  ├───────────────────┼───────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┤
  │ 对齐 gap: 4 bytes │ long 类型要求 8 字节对齐（起始地址必须是 8 的倍数）。Header 12 bytes 后，偏移量 12 不是 8 的倍数，插入 4 bytes gap 让 long 从偏移 16 开始 │
  ├───────────────────┼───────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┤
  │ ★ 总计 24 bytes   │ 比 OneIntObject 多了 8 bytes（4 gap + 4 差异... 实际是 gap 4 + long 比 int 多 4 = 多 8）                                                  │
  └───────────────────┴───────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┘

  5. new MultiFieldObject()（第 89-99 行）

  Mark Word(8) + Class Pointer(4) + int(4) + long(8) + boolean(1) + ref(4) = 29 → 补到 32

  ┌────────────────────────┬───────────────────────────────────────────────────────────────────────────────────┐
  │           值           │                                       含义                                        │
  ├────────────────────────┼───────────────────────────────────────────────────────────────────────────────────┤
  │ int a: 4 bytes         │ 紧跟 Class Pointer                                                                │
  ├────────────────────────┼───────────────────────────────────────────────────────────────────────────────────┤
  │ long b: 8 bytes        │ long 要 8 字节对齐，这里 8+4+4=16 刚好对齐                                        │
  ├────────────────────────┼───────────────────────────────────────────────────────────────────────────────────┤
  │ boolean c: 1 byte      │ 只占 1 字节                                                                       │
  ├────────────────────────┼───────────────────────────────────────────────────────────────────────────────────┤
  │ Object d(ref): 4 bytes │ 压缩指针下引用类型 4 字节                                                         │
  ├────────────────────────┼───────────────────────────────────────────────────────────────────────────────────┤
  │ 小计 29 bytes          │ 不是 8 的倍数                                                                     │
  ├────────────────────────┼───────────────────────────────────────────────────────────────────────────────────┤
  │ 对齐填充 3 bytes       │ 29 + 3 = 32                                                                       │
  ├────────────────────────┼───────────────────────────────────────────────────────────────────────────────────┤
  │ JVM 可能重排字段顺序   │ JVM 会把宽类型（long/double）排前面、窄类型（boolean/byte）排后面，减少中间的 gap │
  └────────────────────────┴───────────────────────────────────────────────────────────────────────────────────┘

  JVM 字段重排规则： long/double > int/float > short/char > byte/boolean > reference，同宽度按声明顺序。父类字段在前、子类字段在后。

  6. new int[10]（第 101-108 行）

  Mark Word(8) + Class Pointer(4) + 数组长度(4) + 数据(10×4=40) = 56 bytes

  ┌───────────────────┬────────────────────────────────────────────┐
  │        值         │                    含义                    │
  ├───────────────────┼────────────────────────────────────────────┤
  │ 数组长度: 4 bytes │ 数组特有，存 length 值。普通对象没有这部分 │
  ├───────────────────┼────────────────────────────────────────────┤
  │ 10 × 4 = 40 bytes │ 10 个 int 元素，每个 4 字节                │
  ├───────────────────┼────────────────────────────────────────────┤
  │ ★ 总计 56 bytes   │ 8+4+4+40=56，刚好是 8 的倍数               │
  ├───────────────────┼────────────────────────────────────────────┤
  │ arr.length = 10   │ 读取的就是对象头中那 4 bytes 的值          │
  └───────────────────┴────────────────────────────────────────────┘

  7. new Child()（第 110-118 行）

  Mark Word(8) + Class Pointer(4) + parentField(4) + childField(4) = 20 → 补到 24

  ┌───────────────────────────────────────────┬────────────────────────────────────────────────────────────────────────────────────┐

  ├───────────────────────────────────────────┼────────────────────────────────────────────────────────────────────────────────────┤

  ├───────────────────────────────────────────┼────────────────────────────────────────────────────────────────────────────────────┤

  对象内存布局日志分析

  一、对象头（Header）

  对象头由 Mark Word + 类型指针 组成（数组多一个长度字段）：

  ┌───────────────┬──────────────────────────┬──────────────────────────────────────────────┐
  │   组成部分    │      64位 JVM 大小       │                     作用                     │
  ├───────────────┼──────────────────────────┼──────────────────────────────────────────────┤
  │ Mark Word     │ 8 bytes                  │ 存 hashCode、GC 年龄、锁状态（随锁状态复用） │
  ├───────────────┼──────────────────────────┼──────────────────────────────────────────────┤
  │ Klass Pointer │ 4 bytes（压缩）/ 8 bytes │ 指向方法区中类的元数据                       │
  ├───────────────┼──────────────────────────┼──────────────────────────────────────────────┤
  │ 数组长度      │ 4 bytes（仅数组）        │ 记录 array.length                            │
  └───────────────┴──────────────────────────┴──────────────────────────────────────────────┘

  Mark Word 复用表（面试重点）：
  - 无锁 → hashCode(31bit) + GC年龄(4bit) + 偏向标志位
  - 偏向锁 → 线程ID(54bit) + epoch(2bit) + GC年龄(4bit)
  - 轻量级锁 → 栈中 Lock Record 指针(62bit)
  - 重量级锁 → Monitor 对象指针(62bit)
  - GC标记 → 空

  关键推论：
  - GC 年龄 4bit → 最大值 2^4-1=15 → 所以 -XX:MaxTenuringThreshold 最大只能设 15
  - 加 synchronized 后 hashCode 被挤走，存到 Monitor 里

  二、7 种对象大小计算

  公式：Mark Word + Klass Pointer + 实例数据 + 对齐填充（补到 8 的倍数）

  ┌─────┬────────────────────────┬─────────────────────────────────────────────────────┬──────┬──────┬──────────┐
  │  #  │          对象          │                       各部分                        │ 小计 │ 填充 │   总计   │
  ├─────┼────────────────────────┼─────────────────────────────────────────────────────┼──────┼──────┼──────────┤
  │ 1   │ new Object()           │ 8 + 4 + 0 = 12                                      │ 12   │ +4   │ 16 bytes │
  ├─────┼────────────────────────┼─────────────────────────────────────────────────────┼──────┼──────┼──────────┤
  │ 2   │ new EmptyObject()      │ 同 Object                                           │ 12   │ +4   │ 16 bytes │
  ├─────┼────────────────────────┼─────────────────────────────────────────────────────┼──────┼──────┼──────────┤
  │ 3   │ new OneIntObject()     │ 8 + 4 + int(4) = 16                                 │ 16   │ 0    │ 16 bytes │
  ├─────┼────────────────────────┼─────────────────────────────────────────────────────┼──────┼──────┼──────────┤
  │ 4   │ new OneLongObject()    │ 8 + 4 + gap(4) + long(8) = 24                       │ 24   │ 0    │ 24 bytes │
  ├─────┼────────────────────────┼─────────────────────────────────────────────────────┼──────┼──────┼──────────┤
  │ 5   │ new MultiFieldObject() │ 8 + 4 + int(4) + long(8) + boolean(1) + ref(4) = 29 │ 29   │ +3   │ 32 bytes │
  ├─────┼────────────────────────┼─────────────────────────────────────────────────────┼──────┼──────┼──────────┤
  │ 6   │ new int[10]            │ 8 + 4 + arrLen(4) + 10×4 = 56                       │ 56   │ 0    │ 56 bytes │
  ├─────┼────────────────────────┼─────────────────────────────────────────────────────┼──────┼──────┼──────────┤
  │ 7   │ new Child()            │ 8 + 4 + parentInt(4) + childInt(4) = 20             │ 20   │ +4   │ 24 bytes │
  └─────┴────────────────────────┴─────────────────────────────────────────────────────┴──────┴──────┴──────────┘

  几个要点：

  OneLongObject 的 gap(4)：long 类型要求 8 字节对齐，Klass Pointer 结束后在第 12 字节，不是 8 的倍数，所以插入 4 bytes 的 gap 再放 long。

  MultiFieldObject 字段重排：JVM 会自动重排字段顺序来减少 padding（比如把 int 和 boolean 放一起），实际布局不一定按代码声明顺序。

  Child 继承：父类字段先排（parentField），子类字段后排（childField），不会混在一起。

  数组对象：比普通对象多 4 bytes 的数组长度字段。
