========================================
  GC Roots 与可达性分析演示
========================================

--- 演示 1：局部变量作为 GC Root ---

  [作用域内] obj 引用 2MB 对象，obj 是 GC Root → 对象存活
  [GC 后] 对象仍存活（obj 还在栈帧中）

  [出作用域] obj 超出作用域
  [GC 后] 注意：槽位未被复用时，对象可能仍未被回收！
  [覆盖槽位后 GC] 现在 obj 的槽位被 placeholder 覆盖 → 对象可被回收
  → 这就是为什么有时需要 obj = null 来帮助 GC

--- 演示 2：静态变量作为 GC Root ---

  staticRef 指向 2MB 对象 → GC Root 可达 → 不会被回收
  staticRef = null → 对象不可达 → 可被回收
  [GC 后] 对象已被回收

--- 演示 3：引用链断裂 ---

  引用链: 栈变量 node1, node2 → 各自的对象
  所有对象通过栈变量（GC Root）可达 → 都存活

  node2 = null → node2 原来指向的对象不可达
  但 node3 仍被栈变量直接引用 → 仍然存活
  [GC 后] node3 数据仍然可访问: 262144 个 int

========================================
  GC Roots 速记（面试必背）：
  1. 虚拟机栈局部变量引用的对象
  2. 方法区静态变量引用的对象
  3. 方法区常量引用的对象
  4. 本地方法栈 JNI 引用的对象
  5. synchronized 持有的对象

  判断对象存活的两种方式：
  × 引用计数法 → 有循环引用问题，JVM 不用
  ✓ 可达性分析 → 从 GC Roots 沿引用链遍历
========================================
