MetaspaceAndDirectOOMDemo 实际运行日志分析（3 个场景）

运行环境：JDK 1.8.0_441, Windows 11

=== 场景一：Metaspace OOM ===

参数：-XX:MaxMetaspaceSize=30m

与编造日志的关键差异：

  ┌──────────────────┬──────────────┬──────────────┐
  │ 数据点           │ 编造值       │ 实际值       │
  ├──────────────────┼──────────────┼──────────────┤
  │ OOM时代理类数量  │ 26783 个     │ 4369 个      │
  │ 5000个时打印     │ 有           │ 没到 5000    │
  │ 错误信息         │ Metaspace    │ Metaspace    │
  └──────────────────┴──────────────┴──────────────┘

  差异分析：
  - 实际只生成了 4369 个代理类，远少于编造的 26783
  - 每个代理类实际占用 Metaspace ≈ 30MB / 4369 ≈ 7KB
  - 编造时按 1KB/类估算，实际代理类包含方法表、字段表等，远不止 1KB
  - 此外每个自定义 ClassLoader 本身也占用 Metaspace（类加载器元数据）

  ★ Bug 发现：代码中 5000 次才打印一次进度，但 4369 就 OOM 了
     → 需要改成 1000 次打印一次才能看到中间过程

=== 场景二：Direct Memory OOM ===

参数：-XX:MaxDirectMemorySize=10m

与编造日志完全一致：

  ┌──────────────────┬──────────────┬──────────────┐
  │ 数据点           │ 编造值       │ 实际值       │
  ├──────────────────┼──────────────┼──────────────┤
  │ OOM时块数        │ 10 块        │ 10 块        │
  │ 累计大小         │ 10.0 MB      │ 10.0 MB      │
  │ 错误信息         │ Direct buffer│ Direct buffer│
  └──────────────────┴──────────────┴──────────────┘

  分析：
  - 每次分配精确 1MB，上限精确 10MB → 行为完全确定
  - 这是唯一一个编造值和实际值完全匹配的场景
  - 因为 ByteBuffer.allocateDirect 的行为是确定性的

=== 场景三：StackOverflowError ===

参数：-Xss256k

与编造日志的关键差异：

  ┌──────────────────┬──────────────┬──────────────┐
  │ 数据点           │ 编造值       │ 实际值       │
  ├──────────────────┼──────────────┼──────────────┤
  │ 递归深度         │ 8427 层      │ 2719 层      │
  │ 每帧大小(推算)   │ 124 字节     │ 96 字节      │
  │ 打印次数         │ 8 次         │ 2 次         │
  └──────────────────┴──────────────┴──────────────┘

  差异分析：
  - -Xss256k（256KB）/ 2719 层 ≈ 96 字节/帧
  - 编造时用的是 1MB 默认栈大小（8427 × 124B ≈ 1MB）
  - 我们用了 -Xss256k，所以栈更小，层数更少
  - 每帧实际 96 字节，比估算的 128-256 范围更小
    （因为 infiniteRecursion() 方法没有局部变量，帧很精简）

  栈帧大小取决于方法复杂度：
  - infiniteRecursion()：无局部变量 → ~96B/帧
  - 有 5 个 int 局部变量的方法 → ~120B/帧
  - 有对象引用+数组的方法 → ~200B/帧

总结 — 编造 vs 实际偏差表：

  ┌───────────────────┬────────────┬────────────┬──────────┐
  │ 场景              │ 编造值     │ 实际值     │ 偏差     │
  ├───────────────────┼────────────┼────────────┼──────────┤
  │ Metaspace 类数量  │ 26,783     │ 4,369      │ 6.1x ↑  │
  │ Direct Memory 块数│ 10         │ 10         │ 精确     │
  │ Stack 递归深度    │ 8,427      │ 2,719      │ 3.1x ↑  │
  └───────────────────┴────────────┴────────────┴──────────┘

  编造时对 Metaspace 每类占用和默认栈大小的假设有误，
  导致 Metaspace 偏差 6 倍、Stack 偏差 3 倍。
  只有 Direct Memory（确定性行为）完全准确。
