========================================
   Spring IoC 容器 refresh() 流程 Demo
========================================

=== 一、AbstractApplicationContext.refresh() 12 步 ===

  ┌─────┬──────────────────────────────────────┬──────────────────────────────────┐
  │ 步骤│ 方法名                                │ 作用                             │
  ├─────┼──────────────────────────────────────┼──────────────────────────────────┤
  │  1  │ prepareRefresh()                     │ 准备工作：设置启动时间、激活标志 │
  │  2  │ obtainFreshBeanFactory()             │ 获取/创建 BeanFactory            │
  │  3  │ prepareBeanFactory(bf)               │ 设置类加载器、SpEL、Aware接口等  │
  │  4  │ postProcessBeanFactory(bf)           │ 子类定制 BeanFactory（模板方法） │
  │  5  │ invokeBeanFactoryPostProcessors(bf)  │ ★ 调用 BeanFactoryPostProcessor │
  │  6  │ registerBeanPostProcessors(bf)       │ 注册 BeanPostProcessor           │
  │  7  │ initMessageSource()                  │ 初始化国际化消息源               │
  │  8  │ initApplicationEventMulticaster()    │ 初始化事件广播器                 │
  │  9  │ onRefresh()                          │ 子类扩展（如 SpringBoot 启动Web）│
  │ 10  │ registerListeners()                  │ 注册事件监听器                   │
  │ 11  │ finishBeanFactoryInitialization(bf)  │ ★ 实例化所有非懒加载的单例 Bean │
  │ 12  │ finishRefresh()                      │ 发布 ContextRefreshedEvent       │
  └─────┴──────────────────────────────────────┴──────────────────────────────────┘

  核心步骤标记 ★：
    第5步：BeanFactoryPostProcessor 可以修改 BeanDefinition（Spring Boot 自动装配的入口）
    第11步：所有单例 Bean 在这里被创建（Bean 生命周期在此触发）

=== 三、BeanFactory vs ApplicationContext ===

  ┌───────────────────┬───────────────────────────┬──────────────────────────────┐
  │ 特性              │ BeanFactory               │ ApplicationContext           │
  ├───────────────────┼───────────────────────────┼──────────────────────────────┤
  │ Bean 实例化时机   │ 延迟加载（getBean时创建） │ 预实例化（启动时创建单例）   │
  │ AOP 支持          │ 手动配置                  │ 自动支持                     │
  │ 事件机制          │ 不支持                    │ 支持（ApplicationEvent）     │
  │ 国际化            │ 不支持                    │ 支持（MessageSource）        │
  │ 资源加载          │ 不支持                    │ 支持（ResourceLoader）       │
  │ 环境抽象          │ 不支持                    │ 支持（Environment）          │
  └───────────────────┴───────────────────────────┴──────────────────────────────┘

  继承关系：
  BeanFactory
    ├── HierarchicalBeanFactory
    ├── ListableBeanFactory
    └── AutowireCapableBeanFactory
          └── DefaultListableBeanFactory（最终实现类）

  ApplicationContext extends BeanFactory
    ├── AnnotationConfigApplicationContext（注解驱动）
    ├── ClassPathXmlApplicationContext（XML驱动）
    └── GenericWebApplicationContext（Web环境）

========================================
   面试速记
========================================

  1. refresh() 是 Spring 容器启动的核心方法，被 AbstractApplicationContext 定义
  2. 12 步流程按顺序：准备 → 获取BF → 定制BF → 后置处理 → 调用BFPP →
     注册BPP → 初始化消息源 → 初始化事件广播 → 刷新子容器 → 注册监听器 →
     实例化单例 → 完成刷新
  3. BeanFactoryPostProcessor 在 Bean 实例化之前执行，可以修改 BeanDefinition
  4. BeanPostProcessor 在 Bean 实例化之后执行，可以包装 Bean（AOP 的入口）
  5. ApplicationContext = BeanFactory + 事件机制 + 国际化 + AOP + 资源加载
