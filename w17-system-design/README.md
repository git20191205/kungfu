## 核心方法论

### 系统设计 4 步法

```
第 1 步：需求澄清（5-10 分钟）
  - 使用 5W1H 方法
  - 确认功能需求和非功能需求
  ↓
第 2 步：容量估算（5 分钟）
  - QPS = DAU × 请求数 / 86400 × 峰值系数
  - 存储 = 单条记录 × 记录数 × 副本数
  ↓
第 3 步：高层架构（10 分钟）
  - 画架构图
  - 说明技术选型
  ↓
第 4 步：深入设计（20 分钟）
  - 数据模型设计
  - 接口设计
  - 核心流程设计
  - 高可用设计
  - 扩展性设计
```

### 秒杀系统案例

基于 w09-seckill 项目的真实数据：

**性能数据**：
- QPS：6711（100 并发）
- P99 延迟：28ms
- 成功率：100%

**核心设计**：
- 四层过滤架构（CDN → Gateway → Redis → Kafka）
- Redis Lua 原子扣库存
- Kafka 异步创建订单
- 三道防线防超卖

**容量估算**：
- 峰值 QPS：40,000（10 万用户，前 10 秒集中）
- 存储：26 KB（100 个订单）
- 带宽：160 Mbps
- 服务器：9 台应用服务器

## 面试准备

### 核心要点

1. **需求澄清**：5W1H、SMART 原则
2. **容量估算**：QPS、存储、带宽、服务器
3. **数据模型**：ER 图、索引、分库分表
4. **接口设计**：RESTful、幂等性、版本控制
5. **高可用**：冗余、限流、熔断、降级
6. **扩展性**：水平扩展、无状态、分片

### 面试话术

- "我先确认一下需求..."
- "让我粗算一下 QPS..."
- "我画一个高层架构图..."
- "为了保证不超卖，我设计三道防线..."
- "系统支持水平扩展，可以从 1 台扩展到 100 台..."

### 常见追问

- "如果 QPS 从 5000 增长到 50000 怎么办？"
- "如何保证数据一致性？"
- "如何防止恶意刷单？"
- "如何从 1 台扩展到 100 台？"
