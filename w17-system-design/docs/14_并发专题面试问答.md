# 并发专题 — 高频面试问答

> 基于 w02-concurrent 项目经验整理的 20 个高频面试题

## 一、线程基础（5 题）

### 1. 线程的生命周期有哪些状态？

**答**：
- **NEW**：新建，未启动
- **RUNNABLE**：可运行（就绪 + 运行）
- **BLOCKED**：阻塞，等待锁
- **WAITING**：等待，wait() / join()
- **TIMED_WAITING**：超时等待，sleep() / wait(timeout)
- **TERMINATED**：终止

### 2. 创建线程有哪几种方式？

**答**：
1. **继承 Thread**：`class MyThread extends Thread`
2. **实现 Runnable**：`class MyTask implements Runnable`（推荐）
3. **实现 Callable**：`class MyTask implements Callable<V>`（有返回值）
4. **线程池**：`ExecutorService pool = Executors.newFixedThreadPool(10)`（推荐）

### 3. sleep() vs wait() 的区别？

**答**：
- **sleep()**：Thread 类方法，不释放锁，休眠指定时间
- **wait()**：Object 类方法，释放锁，等待 notify()

### 4. 如何停止线程？

**答**：
- **interrupt()**：设置中断标志，线程检查 `Thread.interrupted()`
- **volatile 标志位**：`volatile boolean stop = false`
- **不推荐**：stop()（已废弃，不安全）

### 5. 什么是守护线程？

**答**：
- **定义**：后台线程，JVM 退出时自动结束
- **设置**：`thread.setDaemon(true)`（启动前设置）
- **用途**：GC 线程、监控线程

## 二、锁机制（5 题）

### 6. synchronized 的原理是什么？

**答**：
- **对象锁**：每个对象有一个 Monitor（监视器）
- **字节码**：monitorenter / monitorexit
- **锁升级**：无锁 → 偏向锁 → 轻量级锁 → 重量级锁

### 7. ReentrantLock vs synchronized 的区别？

**答**：
| 特性 | synchronized | ReentrantLock |
|------|-------------|---------------|
| 锁类型 | 隐式锁 | 显式锁 |
| 可中断 | 不可中断 | 可中断（lockInterruptibly） |
| 公平锁 | 非公平 | 可选公平/非公平 |
| 条件变量 | 1 个（wait/notify） | 多个（Condition） |
| 性能 | JDK 6 后相当 | 相当 |

### 8. 什么是死锁？如何避免？

**答**：
- **定义**：两个线程互相等待对方释放锁
- **条件**：互斥、持有并等待、不可剥夺、循环等待
- **避免**：
  - 按顺序加锁
  - 使用 tryLock() 超时
  - 使用 Lock 的 tryLock()

### 9. volatile 的作用是什么？

**答**：
- **可见性**：修改立即刷新到主内存，其他线程可见
- **禁止重排序**：内存屏障
- **不保证原子性**：`i++` 不是原子操作

### 10. CAS 的原理是什么？

**答**：
- **定义**：Compare And Swap，比较并交换
- **原理**：`compareAndSet(expect, update)`
- **优点**：无锁，性能高
- **缺点**：ABA 问题、自旋消耗 CPU

## 三、并发工具类（5 题）

### 11. CountDownLatch vs CyclicBarrier 的区别？

**答**：
- **CountDownLatch**：倒计数，一次性，await() 等待计数归零
- **CyclicBarrier**：栅栏，可重用，await() 等待所有线程到达

### 12. Semaphore 的作用是什么？

**答**：
- **定义**：信号量，控制并发数
- **用途**：限流、资源池
- **方法**：acquire() / release()

### 13. ThreadLocal 的原理是什么？

**答**：
- **定义**：线程本地变量，每个线程独立副本
- **原理**：Thread 内部有 ThreadLocalMap
- **注意**：使用后需 remove()，避免内存泄漏

### 14. 线程池的核心参数有哪些？

**答**：
```java
ThreadPoolExecutor(
    int corePoolSize,        // 核心线程数
    int maximumPoolSize,     // 最大线程数
    long keepAliveTime,      // 空闲线程存活时间
    TimeUnit unit,           // 时间单位
    BlockingQueue<Runnable> workQueue,  // 任务队列
    ThreadFactory threadFactory,        // 线程工厂
    RejectedExecutionHandler handler    // 拒绝策略
)
```

### 15. 线程池的拒绝策略有哪些？

**答**：
- **AbortPolicy**：抛异常（默认）
- **CallerRunsPolicy**：调用者线程执行
- **DiscardPolicy**：丢弃任务
- **DiscardOldestPolicy**：丢弃最老任务

## 四、并发容器（5 题）

### 16. ConcurrentHashMap 的原理是什么？

**答**：
- **JDK 7**：Segment 分段锁，16 个 Segment
- **JDK 8**：CAS + synchronized，Node 数组 + 链表/红黑树
- **优点**：并发度高，读不加锁

### 17. CopyOnWriteArrayList 的原理是什么？

**答**：
- **原理**：写时复制，修改时复制新数组
- **优点**：读不加锁，适合读多写少
- **缺点**：内存占用大，数据一致性弱

### 18. BlockingQueue 有哪些实现？

**答**：
- **ArrayBlockingQueue**：数组，有界
- **LinkedBlockingQueue**：链表，可选有界/无界
- **PriorityBlockingQueue**：优先级队列，无界
- **SynchronousQueue**：不存储元素，直接传递

### 19. 什么是 AQS？

**答**：
- **定义**：AbstractQueuedSynchronizer，抽象队列同步器
- **原理**：state 状态 + FIFO 队列
- **应用**：ReentrantLock、Semaphore、CountDownLatch

## 五、总结

并发编程面试核心考点：
1. **线程基础**：生命周期、创建方式、sleep/wait
2. **锁机制**：synchronized、ReentrantLock、死锁、volatile、CAS
3. **并发工具**：CountDownLatch、Semaphore、ThreadLocal、线程池
4. **并发容器**：ConcurrentHashMap、CopyOnWriteArrayList、BlockingQueue、AQS

**面试技巧**：
- 结合项目经验（w09-seckill）
- 说出具体数据（QPS 6711、线程池 10-20）
- 展示优化思路（Redis Lua、布隆过滤器）
