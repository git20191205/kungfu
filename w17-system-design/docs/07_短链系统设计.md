# 短链系统设计

> 经典系统设计题：设计一个 URL 缩短服务（如 bit.ly）

## 一、需求澄清（5W1H）

### 1.1 What - 做什么？

**核心功能**：
- 生成短链：输入长链接，返回短链接
- 短链跳转：访问短链，重定向到长链
- 短链统计：记录访问次数、来源

**示例**：
```
长链接：https://example.com/product/12345?utm_source=ad&utm_medium=email
短链接：https://short.ly/abc123
```

### 1.2 Who - 谁用？

- 营销人员：缩短推广链接
- 开发者：API 集成
- 普通用户：分享链接

### 1.3 When - 什么时候？

- 随时生成短链
- 7×24 小时访问

### 1.4 Where - 在哪里？

- 主要移动端（微信、微博分享）
- PC 端

### 1.5 Why - 为什么？

- 缩短链接长度，方便分享
- 统计点击数据，分析用户行为
- 防止恶意链接，提升安全性

### 1.6 How - 怎么做？

- 用户输入长链接
- 系统生成短链接
- 用户分享短链接
- 其他用户点击短链接
- 系统重定向到长链接

## 二、容量估算

### 2.1 假设条件

- 日活用户：1000 万
- 每个用户平均生成 0.1 个短链（10% 用户生成）
- 每个用户平均点击 2 个短链
- 保存 5 年数据

### 2.2 QPS 估算

**写 QPS（生成短链）**：
```
平均 QPS = 10,000,000 × 0.1 / 86400 = 12 QPS
峰值 QPS = 12 × 3 = 36 QPS
```

**读 QPS（点击短链）**：
```
平均 QPS = 10,000,000 × 2 / 86400 = 231 QPS
峰值 QPS = 231 × 3 = 693 QPS
```

**读写比**：693 / 36 ≈ **20:1**（读多写少）

### 2.3 存储估算

**短链表**：
```
字段：
- id: 8B
- short_code: 8B
- long_url: 200B
- create_time: 8B
- expire_time: 8B
- click_count: 4B
- 索引: 60B

单条记录：300B

每天新增：1,000,000 条
保存 5 年：1,000,000 × 365 × 5 = 1,825,000,000（18 亿）

存储容量 = 300B × 1,825,000,000 × 2（副本）× 1.3（增长）
         = 1.4 TB
```

**分库分表**：
- 单表最大：1000 万条
- 需要分表：1,825,000,000 / 10,000,000 = 183 张表
- 分库：183 / 16 = 12 个库（每库 16 张表）

### 2.4 带宽估算

**跳转接口**：
```
请求大小：150B
响应大小：300B（302 重定向）
峰值 QPS：700

入带宽 = 700 × 150B × 8 = 0.84 Mbps
出带宽 = 700 × 300B × 8 = 1.68 Mbps

需要：2 Mbps（非常小）
```

### 2.5 服务器估算

**应用服务器**：
```
单机 QPS：5000（假设）
需要台数：700 / 5000 = 1 台
考虑冗余：1 × 2 = 2 台
```

**数据库**：
```
需要 12 个 MySQL 实例（分库分表）
```

**缓存**：
```
热点短链（20% 的短链占 80% 的访问）：
- 热点数量：1,825,000,000 × 0.2 = 365,000,000（3.6 亿）
- 单条缓存：300B
- 总缓存：365,000,000 × 300B = 110 GB

Redis Cluster：
- 单机 Redis：16 GB
- 需要节点：110 GB / 16 GB = 7 个节点
- 考虑副本：7 × 2 = 14 个节点（7 主 7 从）
```

### 3.3 核心流程

**生成短链流程**：
```
1. 用户输入长链接
2. 检查是否已存在（MD5 查询）
3. 生成唯一 ID（分布式 ID 生成器）
4. Base62 编码（ID → 短码）
5. 存储到 MySQL
6. 存储到 Redis（热点数据）
7. 返回短链接
```

**短链跳转流程**：
```
1. 用户访问短链接
2. 查询 Redis（热点数据）
3. 如果 Redis 没有，查询 MySQL
4. 302 重定向到长链接
5. 异步发送 Kafka 消息（统计）
6. Kafka 消费者更新点击统计
```

### 4.2 短链生成算法

**方案 1：自增 ID + Base62 编码（推荐）**

```java
public class ShortUrlGenerator {
    
    private static final String BASE62 = "0123456789abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ";
    
    // 1. 生成唯一 ID（分布式 ID 生成器，如 Snowflake）
    public long generateId() {
        return snowflake.nextId();  // 例如：1234567890
    }
    
    // 2. Base62 编码
    public String encode(long id) {
        StringBuilder sb = new StringBuilder();
        while (id > 0) {
            sb.append(BASE62.charAt((int) (id % 62)));
            id /= 62;
        }
        return sb.reverse().toString();  // 例如："abc123"
    }
    
    // 3. Base62 解码
    public long decode(String shortCode) {
        long id = 0;
        for (char c : shortCode.toCharArray()) {
            id = id * 62 + BASE62.indexOf(c);
        }
        return id;
    }
    
    // 4. 生成短链
    public String generateShortUrl(String longUrl) {
        // 检查是否已存在
        String md5 = MD5(longUrl);
        ShortUrl existing = shortUrlMapper.selectByMd5(md5);
        if (existing != null) {
            return existing.getShortCode();
        }
        
        // 生成新短链
        long id = generateId();
        String shortCode = encode(id);
        
        // 存储
        ShortUrl shortUrl = new ShortUrl();
        shortUrl.setId(id);
        shortUrl.setShortCode(shortCode);
        shortUrl.setLongUrl(longUrl);
        shortUrl.setUrlMd5(md5);
        shortUrlMapper.insert(shortUrl);
        
        // 缓存
        redisTemplate.opsForValue().set("short:" + shortCode, longUrl, 7, TimeUnit.DAYS);
        
        return shortCode;
    }
}
```

**优点**：
- 短链长度固定（6-8 位）
- 生成速度快
- 无冲突

**方案 2：Hash + 冲突检测**

```java
public String generateShortCode(String longUrl) {
    String hash = MD5(longUrl).substring(0, 6);  // 取前 6 位
    
    // 检查冲突
    while (shortUrlMapper.existsByShortCode(hash)) {
        hash = MD5(longUrl + System.nanoTime()).substring(0, 6);  // 加盐重试
    }
    
    return hash;
}
```

**缺点**：
- 可能冲突，需要重试
- 性能较差

```java
@GetMapping("/{shortCode}")
public void redirect(@PathVariable String shortCode, HttpServletResponse response) {
    // 1. 查询 Redis（热点数据）
    String longUrl = redisTemplate.opsForValue().get("short:" + shortCode);
    
    if (longUrl == null) {
        // 2. 查询 MySQL
        ShortUrl shortUrl = shortUrlService.getByShortCode(shortCode);
        if (shortUrl == null) {
            throw new NotFoundException("短链不存在");
        }
        longUrl = shortUrl.getLongUrl();
        
        // 3. 回写 Redis
        redisTemplate.opsForValue().set("short:" + shortCode, longUrl, 7, TimeUnit.DAYS);
    }
    
    // 4. 302 重定向
    response.setStatus(HttpServletResponse.SC_FOUND);
    response.setHeader("Location", longUrl);
    
    // 5. 异步统计
    kafkaTemplate.send("short-url-click", new ClickEvent(shortCode));
}
```

@Component
public class ShortUrlBloomFilter {
    
    private BloomFilter<String> bloomFilter;
    
    @PostConstruct
    public void init() {
        // 预期 18 亿短链，误判率 1%
        bloomFilter = BloomFilter.create(
            Funnels.stringFunnel(Charset.defaultCharset()),
            1_800_000_000L,
            0.01
        );
        
        // 加载已有短链到布隆过滤器
        loadExistingShortCodes();

    public boolean mightExist(String shortCode) {
        return bloomFilter.mightContain(shortCode);
    }
    
    public void add(String shortCode) {

}
