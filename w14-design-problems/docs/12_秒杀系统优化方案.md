## 二、优化路线图

### 阶段 1：短期优化（1 周，成本 ¥0）

**目标**：QPS 6711 → 8000

**优化项**：

1. **JVM 参数调优**
```bash
# 使用 G1GC，固定堆大小
JAVA_OPTS="-Xms8g -Xmx8g -XX:+UseG1GC -XX:MaxGCPauseMillis=50"
```

2. **连接池预热**
```java
@Component
public class ConnectionPoolWarmup implements ApplicationRunner {
    @Override
    public void run(ApplicationArguments args) {
        // Redis 连接池预热
        ExecutorService pool = Executors.newFixedThreadPool(50);
        for (int i = 0; i < 50; i++) {
            pool.submit(() -> redisTemplate.opsForValue().get("warmup"));
        }
        
        // Kafka 连接池预热
        for (int i = 0; i < 10; i++) {
            kafkaTemplate.send("warmup", "test");
        }
    }
}
```

3. **HTTP Keep-Alive**
```yaml
server:
  tomcat:
    keep-alive-timeout: 60000
    max-keep-alive-requests: 100
```

**预期效果**：QPS 8000，提升 19%

### 3.2 网关优化

**动态限流**：
```java
@Component
public class DynamicRateLimiter {
    
    // 根据系统负载动态调整限流阈值
    @Scheduled(fixedRate = 10000)
    public void adjustRateLimit() {
        double cpuUsage = getCpuUsage();
        
        if (cpuUsage > 80) {
            // CPU 使用率 > 80%，降低限流阈值
            rateLimiter.setRate(3000);
        } else if (cpuUsage < 50) {
            // CPU 使用率 < 50%，提高限流阈值
            rateLimiter.setRate(6000);
        }
    }
}
```

