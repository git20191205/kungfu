# 分布式缓存系统设计

> 经典系统设计题：设计 Redis Cluster 分布式缓存系统

## 一、需求澄清

**核心功能**：
- 数据存储：KV 存储，支持多种数据结构
- 高性能：QPS 10 万+，P99 < 10ms
- 高可用：99.99% SLA，自动故障转移
- 可扩展：支持水平扩展

**非功能需求**：
- 数据量：1 TB
- QPS：100,000
- 延迟：P99 < 10ms
- 可用性：99.99%

## 二、容量估算

**QPS 估算**：100,000（读多写少 80:20）
**存储估算**：1 TB
**服务器估算**：
- 单机 Redis：QPS 10 万，存储 16 GB
- 需要节点：1 TB / 16 GB = 64 个节点
- 考虑副本：64 × 2 = 128 个节点（64 主 64 从）

## 三、高层架构

### 3.1 架构图

```
客户端
  ↓
┌──────────────────────────────────┐
│      Redis Cluster               │
├──────────────────────────────────┤
│  Master1 (slot 0-2730)    Slave1 │
│  Master2 (slot 2731-5460) Slave2 │
│  ...                             │
│  Master64 (slot ...)      Slave64│
└──────────────────────────────────┘

分片算法：CRC16(key) % 16384
```

### 3.2 核心设计

**一致性 Hash**：
```
优点：增删节点影响小
缺点：数据分布不均（需要虚拟节点）

Redis Cluster 使用 Hash Slot：
- 16384 个 slot
- CRC16(key) % 16384 计算 slot
- 每个 Master 负责一部分 slot
```

**主从复制**：
```
Master → Slave 异步复制
Slave 只读
Master 故障，Slave 自动提升
```

**故障检测**：
```
心跳检测：每秒 PING
超时判定：5 秒未响应标记为 PFAIL
投票选举：半数以上节点同意标记为 FAIL
自动切换：Slave 提升为 Master
```

## 四、深入设计

### 4.1 数据分片

**Hash Slot 分配**：
```bash
# 创建集群（3 主 3 从）
redis-cli --cluster create \
  192.168.1.1:6379 192.168.1.2:6379 192.168.1.3:6379 \
  192.168.1.4:6379 192.168.1.5:6379 192.168.1.6:6379 \
  --cluster-replicas 1

# Slot 分配：
# Master1: 0-5460
# Master2: 5461-10922
# Master3: 10923-16383
```

**路由算法**：
```java
public String route(String key) {
    int slot = CRC16.crc16(key) % 16384;
    // 查询 slot 对应的节点
    return clusterNodes.get(slot);
}
```

### 4.2 高可用设计

**主从切换**：
```
1. Master 故障（心跳超时）
2. Slave 发起选举
3. 投票选出新 Master
4. 新 Master 接管 slot
5. 通知客户端更新路由
```

**脑裂问题**：
```
配置：
min-replicas-to-write 1  # 至少 1 个 Slave 才能写入
min-replicas-max-lag 10  # Slave 延迟 < 10 秒

防止脑裂导致数据丢失
```

### 4.3 性能优化

**Pipeline**：
```java
// 批量操作，减少网络往返
Jedis jedis = new Jedis("localhost", 6379);
Pipeline pipeline = jedis.pipelined();
for (int i = 0; i < 1000; i++) {
    pipeline.set("key" + i, "value" + i);
}
pipeline.sync();
```

**本地缓存**：
```java
// L1: 本地缓存（Caffeine）
// L2: Redis

LoadingCache<String, String> cache = Caffeine.newBuilder()
    .maximumSize(10000)
    .expireAfterWrite(5, TimeUnit.MINUTES)
    .build(key -> redisTemplate.opsForValue().get(key));
```

### 4.4 扩展性设计

**扩容步骤**：
```bash
# 1. 添加新节点
redis-cli --cluster add-node 192.168.1.7:6379 192.168.1.1:6379

# 2. 重新分片（迁移 slot）
redis-cli --cluster reshard 192.168.1.1:6379

# 3. 添加 Slave
redis-cli --cluster add-node 192.168.1.8:6379 192.168.1.7:6379 --cluster-slave
```
