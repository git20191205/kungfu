# 文件存储系统设计

> 经典系统设计题：设计类似 OSS 的对象存储系统

## 一、需求澄清

**核心功能**：
- 文件上传：支持大文件（GB 级）
- 文件下载：支持断点续传
- 文件删除：软删除 + 定期清理
- 文件访问：支持公开/私有访问

**非功能需求**：
- 存储容量：PB 级
- QPS：10,000（读多写少 90:10）
- 可用性：99.99%
- 持久性：99.999999999%（11 个 9）

## 二、容量估算

**存储估算**：
```
日活用户：1000 万
每个用户每天上传 1 个文件（平均 10 MB）
每天新增：10,000,000 × 10 MB = 100 TB
保存 5 年：100 TB × 365 × 5 = 182 PB
```

**QPS 估算**：
```
上传 QPS = 10,000,000 / 86400 = 116 QPS
下载 QPS = 116 × 9 = 1044 QPS
总 QPS = 1160
```

**服务器估算**：
```
存储服务器：182 PB / 10 TB = 18,200 台
应用服务器：1160 / 5000 = 1 台（考虑冗余 3 台）
```

### 4.2 文件上传

**小文件上传（< 100 MB）**：
```java
@PostMapping("/upload")
public Result<String> upload(@RequestParam MultipartFile file) {
    // 1. 生成文件 ID
    long fileId = snowflake.nextId();
    
    // 2. 计算存储路径
    String path = generatePath(fileId);  // /2026/04/27/fileId
    
    // 3. 上传到分布式文件系统
    hdfsClient.upload(file.getInputStream(), path);
    
    // 4. 保存元数据
    FileMeta meta = new FileMeta();
    meta.setId(fileId);
    meta.setFileName(file.getOriginalFilename());
    meta.setStoragePath(path);
    fileMetaMapper.insert(meta);
    
    // 5. 返回文件 URL
    return Result.success("https://cdn.example.com/" + fileId);
}
```

**大文件上传（> 100 MB）- 分片上传**：
```java
// 1. 初始化分片上传
@PostMapping("/upload/init")
public Result<String> initUpload(@RequestBody InitUploadRequest request) {
    String uploadId = UUID.randomUUID().toString();
    redisTemplate.opsForValue().set("upload:" + uploadId, request, 1, TimeUnit.HOURS);
    return Result.success(uploadId);
}

// 2. 上传分片
@PostMapping("/upload/part")
public Result<Void> uploadPart(@RequestParam String uploadId,
                                @RequestParam Integer partNumber,
                                @RequestParam MultipartFile file) {
    String path = "/tmp/" + uploadId + "/part" + partNumber;
    hdfsClient.upload(file.getInputStream(), path);
    return Result.success();
}

// 3. 完成上传（合并分片）
@PostMapping("/upload/complete")
public Result<String> completeUpload(@RequestParam String uploadId) {
    // 合并分片
    String finalPath = mergeParts(uploadId);
    
    // 保存元数据
    FileMeta meta = new FileMeta();
    meta.setStoragePath(finalPath);
    fileMetaMapper.insert(meta);
    
    return Result.success(meta.getId().toString());
}
```

### 4.3 文件下载

**普通下载**：
```java
@GetMapping("/download/{fileId}")
public void download(@PathVariable Long fileId, HttpServletResponse response) {
    // 1. 查询元数据
    FileMeta meta = fileMetaMapper.selectById(fileId);
    
    // 2. 从分布式文件系统读取
    InputStream inputStream = hdfsClient.read(meta.getStoragePath());
    
    // 3. 写入响应
    response.setContentType("application/octet-stream");
    response.setHeader("Content-Disposition", "attachment; filename=" + meta.getFileName());
    IOUtils.copy(inputStream, response.getOutputStream());
}
```

**断点续传**：
```java
@GetMapping("/download/{fileId}")
public void download(@PathVariable Long fileId, 
                     @RequestHeader(value = "Range", required = false) String range,
                     HttpServletResponse response) {
    FileMeta meta = fileMetaMapper.selectById(fileId);
    
    if (range != null) {
        // 解析 Range: bytes=0-1023
        long start = parseStart(range);
        long end = parseEnd(range);
        
        // 206 Partial Content
        response.setStatus(HttpServletResponse.SC_PARTIAL_CONTENT);
        response.setHeader("Content-Range", "bytes " + start + "-" + end + "/" + meta.getFileSize());
        
        // 读取指定范围
        InputStream inputStream = hdfsClient.read(meta.getStoragePath(), start, end);
        IOUtils.copy(inputStream, response.getOutputStream());
    } else {
        // 正常下载
        // ...
    }
}
```
