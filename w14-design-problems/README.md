## 设计亮点

### 短链系统
- 短链生成：Snowflake ID + Base62 编码
- 读写比：20:1（读多写少）
- 分库分表：12 个库 × 16 张表 = 192 张表
- ���点缓存：Redis 缓存 20% 短链（占 80% 访问）

### 消息推送系统
- 连接管理：Netty + Redis 存储在线状态
- 百万连接：100 台服务器，每台 1 万连接
- 跨服务器推送：Redis Pub/Sub + RPC
- 高可用：心跳机制 + 自动重连 + 消息确认

### Feed 流系统
- 推拉结合：普通用户推模式，大 V 拉模式
- 时间线存储：Redis ZSet（score 是时间戳）
- 异步推送：Kafka 异步写入粉丝 Feed
- 读写比：10:1

### 秒杀系统优化
- 短期（1周，¥0）：JVM 调优 → QPS 8000
- 中期（1月，8台）：分机部署 → QPS 15000
- 长期（3月，28台）：K8s 集群 → QPS 30000+
